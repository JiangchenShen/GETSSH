## 🌐 全协议总线级进化与智能识别 (Multi-Protocol Convergence & Smart Detection)

我们彻底打通了本地、远端与物理路网的数据交换链路，让 Xterm.js 成为真正的全能数据渲染核心。
* **本地终端 (Local CLI) 免编译接入：** 摒弃重型交叉编译依赖，全面引入支持多平台 Electron 预编译的原生 PTY 引擎（`@homebridge/node-pty-prebuilt-multiarch`）。现在您可以在应用内秒级拉起本地命令行 Tab！macOS 环境下直通原生 `Zsh/Bash`，Windows 环境下原生唤醒 `PowerShell/CMD`，物理打包风扇不再咆哮。
* **网络交换机与路网设备 (Telnet) 吞噬：** 完美接管 Telnet 协议流，全面支持远程调试思科 Cisco IOS、华为 VRP、新华三 H3C 等硬件路由交换设备。针对路网硬件终端，底层自动锁定 `vt100` 色彩降维兼容方案，完美拦截并处理特有的 `--More--` 分页行为。
* **秒级智能协议自适应解析：** 在新建连接时，前端正则引擎将在您输入主机地址时在后台动态捕获。无论是复制了复合连接串（如 `ssh://admin@host:port`）还是内网硬件特写（如 `192.168.1.1:23`），系统将毫秒级自动切换对应协议并隐显、回填表单，做到“所填即所选”。
* **双轨制手动锁定与作用域隔离：** 自动识别引擎**仅在“创建新终端”时动态激活**。一旦用户从侧边栏点击已保存的旧会话，或者主动在表单中手动点选了协议，系统会立刻切断自动识别，无条件锁定并还原您存储的专属配置，绝不自作聪明。
* **科普级新手提示 (Info Tooltip)：** 在协议切换框旁新增冷酷直角信息图标（`<Info size={14} />`）。鼠标悬停时，将弹出中英双语的网工级协议科普直角气泡，为新手小白提供最温柔的破冰关怀。

## 🎨 极客表现力与侧边栏美学重塑 (Terminal Aesthetics & Sidebar Badge)

* **高冷单色线框 OS 徽章：** 我们消灭了高饱和度、滑稽突兀的原厂彩色 Logo！侧边栏服务器卡片全面重构为**“直角线框限定徽章”**。默认状态下，图标呈现内敛的半透明去色冷灰态（`filter saturate-0 opacity-40 brightness-150`），完美融入黑曜石界面；只有在**鼠标悬停或会话激活**时，才会克制、隐约地透出原厂品牌色，作为极其高级的交互视觉反馈。
* **尺寸提权与防挤压形变：** 抛弃过小的图标尺寸，全量固化为标准的 `w-[18px] h-[18px]` 框架，并注入 `shrink-0` 属性。现在无论您的服务器别名有多长、即使触发了截断省略号，OS 徽章也将如同磐石一般立在左侧，绝不发生任何物理形变。
* **动态内边距 (Padding) 热重载：** 高级终端设置中正式支持 `0px` 到 `32px` 的物理间距自由无级调节。配合 `requestAnimationFrame` 防死锁渲染流，完美避免窗口缩放引发的浏览器内核崩溃，重塑大屏排版的呼吸感。
* **200ms 工业级视觉响铃 (Visual Bell)：** 当配置为视觉响铃时，高频报错不再触发刺耳的系统蜂鸣，而是通过一个在 Canvas 顶层一闪而过的 `bg-white/20` 高性能遮罩层进行温柔警示，200ms 渐变淡出，不占多余内存。

## ⌨️ 跨平台原生习惯对齐 (Cross-Platform Optimization)

* **Windows 专属右键神级复刻：** 彻底终结跨平台运维的肌肉记忆冲突！在 Windows 环境下，系统将默认开启极客崇拜的 PuTTY 经典模式——物理拦截墨迹的上下文二次菜单，**选区内有高亮高显则右键自动复制并清空选择，无高亮则零延迟读取系统剪贴板直接写入 PTY 流**。鼠标一点，命令瞬间执行。

## 🛡️ 安全审计与阻断级漏洞歼灭 (Audit & Critical Fixes)

* **500 条限额会话审计日志 (Connection Logs)：** 上线企业级会话审计面板，白纸黑字记录“连了谁、几点连、连了多久”。**【核心安全红线】坚决不记录、不捕获任何用户的输入命令与代码明文，防止任何密码和敏感令牌泄露！** 底层通过 `.slice(-500)` 机制强制锁定容量，彻底绝育本地 JSON 膨胀导致的 I/O 锁死或内存泄漏风险，支持一键导出为标准 CSV/JSON 报表。
* **物理消灭 `::1:22` 连接报错：** 深度重构了前端连接执行器的中央路由调度器。当触发连接时，系统会严格根据 `session.protocol` 进行全量分支路由，彻底掐死了本地终端（Local CLI）被盲目误判为远端 SSH 从而连接本地 22 端口遭拒的低级逻辑阻断。
* **放开 Local Schema 保存限制：** 优化了状态落盘的 Validation 校验逻辑。当协议为 Local CLI 时，自动跳过 Host、Username 和 Password 的必填锁，使纯净的本地终端卡片能在侧边栏平稳常驻。

## 🔮 Roadmap 前瞻：v1.4.0 Workspace（工作区）控制台与星辰大海

随着 v1.3.1 成功将协议流彻底打通，在接下来的 **v1.4.0 版本**中，GETSSH 将迎来产品形态上的又一次降维打击 —— 推出**全局工作区 (Workspace) 控制台**。

我们将允许用户摆脱传统的单一服务器树状列表，以最高一级容器的视野对复杂运维场景进行“项目级/客户级”的宏观隔离：
* **上下文物理隔离 (Context Grouping)：** 切换工作区，侧边栏会话、密钥管理器、甚至是审计日志瞬间全量对调。您可以在“生产环境工作区”置顶一条冷酷的暗红边框警示，在“开发环境工作区”挂载冷绿边框，彻底告别脑抽误操作。
* **窗体布局快照 (Layout Snapshot)：** 像 `tmux` 或高级 IDE 一样，支持一键保存当前的分屏布局状态。上班打开工作区，系统瞬间自动并发连接多台指定服务器，并完美复活 1/4 分屏、左右分屏的工作战场。
* **工作区局部变量与插件沙盒：** 允许为特定工作区注入专属的命令宏面板与局部插件，让特定的生态工具只在特定的上下文沙盒内生效。

## 🍏 macOS 用户专属：绕过 Apple Gatekeeper 物理拦截指南 (Gatekeeper Bypass)

由于独立开源项目在打包分发时未向 Apple 缴纳“保护费”（未购买官方开发者证书签名），macOS 默认的恶意软件防护机制（Gatekeeper）会在您首次双击打开时进行拦截，弹出“无法验证开发者”或“应用已损坏，应将其移至废纸篓”的断裂级警告。

这属于正常安全策略，请使用以下两套极客手段轻松绕过、平稳通关：

### 🛠️ 流派一：右键破防与系统设置过载 (图形界面通关)
* **硬核右键开门**：下载并解压（或挂载 DMG 拖入 Applications）后，**切勿直接双击**。请在 Finder 中找到 `GETSSH.app`，**按住 Control 键的同时点击鼠标右键**，在弹出的上下文菜单中选择 **“打开 (Open)”**。在随后跳出的警告弹窗中，系统会破例额外暴露出一个 **“打开”** 按钮，点击即可永久写入信任链。
* **安全性面板放行**：如果上述操作未成功，请直接打开系统的 **“系统设置 (System Settings)” -> “隐私与安全性 (Privacy & Security)”**，一路向下滑动找到“安全性”核心板块。此时您会看到一行专属提示：*“已阻止使用 GETSSH.app，因为来自身体不明的开发者”*。直接无畏地点击下方的 **“仍要打开 (Open Anyway)”** 并验证您的 Mac 锁屏密码/指纹，即可满血解锁。

### 💻 流派二：终端一行流属性剥离 (彻底歼灭“应用已损坏”红字死锁)
如果您将应用拖入系统的 `Applications (应用程序)` 目录后，双击依然强行报错提示“应用已损坏，无法打开”，这是苹果系统对未签名资产强行施加的本地隔离属性（Quarantine）在作祟。

请直接打开您 Mac 本地的终端（Terminal），复制并敲入以下指令：

```bash
sudo xattr -rd com.apple.quarantine /Applications/GETSSH.app
```

## ⚠️ 终极预警：macOS Intel 架构与 Rosetta 2 步入最后停服倒计时

始终站在技术前沿的开发者们请注意，Apple 官方已正式通告了其长达数年的架构迁移终局时间线。这一变动将直接影响所有 Mac 用户及跨平台编译生态：

1. **硬件退役：** 当前的 **macOS 26 (Tahoe)** 已被官方确认为**最后一届支持 Intel 芯片 Mac 硬件的操作系统**。未来的 macOS 27 及后续系统将彻底转为 Apple Silicon (M系列芯片) 独占，老款 Intel Mac 将无缘新版系统更新。
2. **转译终结：** 尽管即将到来的 macOS 27 将继续保留 Rosetta 2 运行时，以允许 Apple Silicon 用户过渡运行传统的 x64 应用；但 Apple 已明确通告，**将于 2027 年底发布的 macOS 28 中全面移除/停用通用的 Rosetta 2 架构转译层**。届时，所有未做原生 ARM 适配的 Intel 架构二进制程序，在最新的 macOS 系统上将**彻底无法打开**。

**GETSSH 官方重要通知：**
* **尽早完成物理迁移：** 我们强烈建议所有还在使用老旧 Intel 机型的极客，尽早将物理设备升级至 **Apple Silicon (M系列 ARM 架构)**，以获得最极限的性能、完整的协议总线以及未来的高扩展插件沙盒体验。
* **停更倒计时常态化：** 随着上游各类底层编译依赖库、Chromium 容器以及 Node.js 运行时逐步切断对 x86-64 架构在 macOS 平台上的交叉编译支持，GETSSH 将在维持基本稳定性的前提下进行最后的生命周期维护。**一旦上游开源链彻底断供，GETSSH 将同步下线并永久停止 macOS Intel (x64) 独立安装包的编译与打包分发**。
