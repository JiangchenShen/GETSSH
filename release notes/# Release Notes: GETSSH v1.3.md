## 🔌 插件沙盒生态系统正式贯通 (Plugin Sandbox Ecosystem)

这是 GETSSH 迈向终极形态的最核心跨越。我们抛弃了硬编码，彻底打通了从本地文件物理提取到动态沙盒渲染的全链路闭环。
* **动态挂载与解析引擎：** 系统现已支持动态侦测 `plugins` 目录下的解压资产。智能解析插件内部的 `package.json`（优先读取 `getssh.name` / `displayName` 并优雅降级至 `name` 字段），结合 Zustand 状态树实现插件卡片的动态按需渲染。
* **终结 macOS 物理路径“黑洞”：** 精准排查并修复了 macOS 环境下极度隐蔽的 Chromium 底层拦截机制。针对 `app.getPath('userData')` 因包含系统级空格导致本地 `file://` 协议解析断裂的致命缺陷，现已通过标准 `encodeURI` 强制物理路径转码，完美打通本地安全路由。
* **Iframe 进程绝对隔离：** 对插件渲染层进行精细的安全管控，注入 `sandbox="allow-scripts allow-same-origin"` 指令，在确保插件内 JS 引擎正常运转的同时，阻断对外部敏感环境的越权访问。
* **首发官方插件 - System Monitor：** 成功实装“本地系统资源监控”插件。以极致的紫黑赛博朋克 UI，实时抓取、可视化渲染本机 CPU 核心调度流水线与内存的极限负载状态。

## 🎨 UI 视觉重构与交互补全 (Visual & Interaction)

* **浅色模式终端“失明”紧急抢修：** 扑灭了因全局注入高对比度深色文本而引发的级联 CSS 越权污染危机。现已为 `xterm.js` 实例穿上“防化服”，通过显式配置 `theme: { foreground: '#FFFFFF' }` 强制锁死 Canvas 前景色，彻底捍卫 SSH 交互区的极客体验。
* **极致深色毛玻璃 (Cyberpunk Glassmorphism)：** 全面重构深色模式的混合逻辑，摒弃多余色值干扰，呈现纯净的黑曜石通透质感。
* **材质逻辑降级与剥离：** 在浅色模式下果断弃用毛玻璃特效，退回高对比度的实体底色 (`bg-slate-50`)，确保强光环境下的绝对可读性。
* **全新交互 - 右键智能剪贴：** 正式引入终端区域右键自动复制/粘贴功能，大幅缩短高频操作的链路，让运维与敲命令的体验如同行云流水。
* **资源路由修复：** 修复了 Vite 生产包因静态资源重定向导致左上角 Logo SVG 图标丢失的问题。

## 🛡️ 代码健壮性与极致压榨 (Type Safety & Performance)

* **TypeScript `any` 灭绝计划：** 开展了极其严格的类型系统重构，彻底消灭了 `SSHConnectConfig` 及面板注入中散落的 `any` 动态类型，实现主渲染进程间 100% 强类型安全传递。
* **极致的微秒级性能压榨：** 采用 `Promise.all` 实现跨进程配置的并发解析加载。优化 `svgSanitizer` 和状态树遍历的数组分配，采用 $O(1)$ 级别的极速过滤，大幅降低内存与 GC 抖动。为插件沙盒增加 Renderer 脚本内存级缓存，消除 V8 重复编译开销。
* **TDD 测试覆盖网：** 全面引入 Vitest 单元测试，为核心状态树与组件铺设了高密度的自动化测试用例。
* **底层日志降噪：** 修复了极端断线场景下的空白回调报错以及剪贴板的未捕获异常隐患，并清除了控制台海量的心跳轮询日志噪音。

## 🏭 极客工程化与分发体系 (Engineering Hub)

* **Apple 级构建追踪：** 正式演进版本号命名规范，附加五位字母数字混合的构建码（**Build B7X9Q**），并注入 UI，大幅增强发布包的防篡改检验与溯源能力。

## 🔮 Roadmap 前瞻：v1.4.0 开发者模式与“权限梯队”架构

v1.3.0 成功筑起了坚不可摧的底层沙盒围墙，而在即将到来的 **v1.4.0 版本**中，我们将正式开启“受控的城门”——推出完整的**开发者模式 (Developer Mode)** 与核心 SDK API。

为了在“极客自由”与“系统安全”之间取得完美平衡，v1.4.0 将引入一套严密的**阶梯式权限体系 (Permission Tiers)**，为全球开发者提供标准化的插件开发闭环：

### 1. 基础调试赋能 (Debugging & Workflow)
* **免打包加载 (Load Unpacked)：** 开发者无需频繁打包 `.zip`，可直接指定本地工程目录进行动态挂载与调试。
* **独立 DevTools 唤醒：** 允许为单一插件的 Iframe 容器独立唤出 Chrome 开发者工具，实现精准的 DOM、CSS 与 Console 报错追踪。
* **动态热重载 (Hot Reload)：** 支持监听本地工作区文件变化，实现插件界面的毫秒级热刷新。

### 2. 核心业务 API 开放 (Core SDK Capabilities)
插件需要在 `package.json` 中显式声明所需权限，方可调用 GETSSH 暴露的底层桥接接口：
* **`ui:sidebar` (UI 挂载权)：** 允许插件脱离基础的 Welcome 页面，在应用左侧边栏注册专属 Icon，并接管主工作区的完整渲染逻辑。
* **`session:read` (会话读取权)：** 允许插件安全地获取当前活跃的 SSH 标签页状态与主机基础信息（严格屏蔽密码与私钥）。
* **`terminal:read/write` (终端流控权)：** 最核心的生产力接口。允许高级插件监听当前活跃终端的 `stdout` 输出流，或向终端静默注入并执行特定命令（例如：自动化装机脚本面板、报错日志自动捕获）。

### 3. 宿主与物理环境特权 (Host-Level Privileges)
针对极客级的底层需求，我们将开放突破默认沙盒的受控特权：
* **`host:exec` (命令执行权)：** 允许插件通过受控的 `child_process` 模块，在用户的物理机（Mac/Windows/Linux）上执行特定的系统级查询命令。
* **`fs:app-data` (专属持久化读写权)：** 允许插件在自身的沙盒目录（UserData）内创建配置文件或 SQLite 数据库，实现复杂业务数据的安全持久化。

**安全承诺：** 任何涉及上述核心及特权的插件，在终端用户安装时，GETSSH 均会弹出清晰的“权限请求清单 (Permission Consent)”。我们将把系统的绝对控制权，永远交还给用户。

准备好您的代码编辑器，GETSSH 的高扩展插件生态即将迎来大爆发！

## ⚠️ 终极预警：macOS Intel 架构即将进入生命周期倒数

随着 Apple 官方生态的全面进化，**苹果将于一年后的 macOS 28 正式结束对 x86 (Intel 芯片) 机型的底层支持（Rosetta 2 即将退役）。**

这意味着，GETSSH 在后续升级各类底层编译库、Node.js 运行时及插件通信桥接时，将越来越难以维持在 macOS Intel 架构上的运行稳定性。
**重要通知：**
1. 我们强烈建议所有还在使用 Intel 款 Mac 的极客开发者，尽早将设备升级至 **Apple Silicon (M系列 ARM 架构)**，以获得最极限的性能与最完整的插件生态体验。
2. 当我们因上游底层库彻底断供，导致无法继续维护 x86 版本时，GETSSH 官方将会发布正式通告，并**永久停止 macOS Intel 版本的更新与打包**。
