# GETSSH — 产品需求文档 (PRD)

**文档版本**：V1.0（反向推导版）
**产品名称**：GETSSH
**当前版本**：v1.3.2-preview
**文档性质**：本文档为基于现有代码库进行反向工程推导的产品需求文档，忠实还原了产品的核心功能边界、技术决策与设计哲学。

---

## 1. 产品概述

### 1.1 产品定位

GETSSH 是一款面向专业开发者与运维工程师的**跨平台桌面 SSH 客户端**，以 Electron + React 构建，主打「极简极客」美学与「生产级」安全防护的双重体验。

- **竞品对标**：Terminus, Royal TSX, SecureCRT
- **核心差异化**：原生毛玻璃（Glassmorphism）UI + 运行时应用自我保护（RASP）沙盒 + 类 VSCode 分屏架构 + 可扩展插件生态

### 1.2 目标用户

| 用户画像 | 描述 |
|---|---|
| **主力用户** | 需要频繁管理多台远端 Linux/Unix 服务器的后端工程师、DevOps 工程师 |
| **次要用户** | 对 SSH 工具有安全敏感性要求的 Web3/安全研究人员 |
| **潜在用户** | 对替换 iTerm/Terminal 有意愿但希望集成化体验的全栈开发者 |

### 1.3 平台目标

- **MVP**：macOS (arm64 Apple Silicon 优先)
- **规划中**：Windows (x64), Linux (AppImage)

## 2. 核心功能模块

### 2.1 SSH 连接管理

**目标**：让用户能够安全、快速地管理多个 SSH 会话。

| 功能 | 优先级 | 描述 |
|---|---|---|
| 快速连接 (Quick Connect) | P0 | 在 Command Center 搜索框输入 `user@host`，按 Enter 即发起连接 |
| 会话配置 (Session Profiles) | P0 | 支持设置 `alias`、`host`、`username`、`port`、`password`、`privateKeyPath`、`protocol` |
| 标签页管理 (Tab Bar) | P0 | 多标签，支持自定义标题，状态实时同步 |
| 自动连接 (Auto-Start) | P1 | 用户可标记特定主机，应用启动后自动建立隧道 |
| SSH Keepalive | P1 | 全局可配置心跳间隔，防止 NAT 超时断线 |
| 主机密钥验证 (Host Key) | P1 | 首次连接时弹出 `HostKeyVerificationModal`，防止中间人攻击 |
| 代理支持 (Proxy) | P2 | 支持全局配置 SOCKS/HTTP 代理转发 |
| 初始化脚本 (initScript) | P2 | 连接成功后自动向远端发送预设命令（如 `tmux`, `zsh`） |

**技术约束**：
- 底层使用 `ssh2` Node.js 库
- 会话 ID 使用 `crypto.randomUUID()` 生成，防止顺序猜测
- PTY 终端尺寸须进行合理 Clamp，防止 C++ 层缓冲区崩溃

### 2.2 终端体验

**目标**：提供接近原生终端的流畅体验，同时具备分屏、主题等高级功能。

| 功能 | 优先级 | 描述 |
|---|---|---|
| xterm.js 渲染引擎 | P0 | 支持 WebGL 加速渲染，字体连字支持（Ligatures Addon）|
| 分屏 (Split Pane) | P0 | 支持水平分割（hsplit）与垂直分割（vsplit），任意深度嵌套 |
| 主题系统 | P1 | 内置多套主题（暗色/浅色），支持用户导入自定义 JSON 主题文件 |
| 全局字体配置 | P1 | 可配置字体家族和字体大小 |
| 搜索功能 | P2 | 终端内容全文搜索 |
| Unicode 完整支持 | P2 | 支持 Emoji、CJK 字符正确渲染 |

**关键技术决策**：
- 采用「CSS 常驻挂载」策略，切换标签时终端进程不被销毁（保活状态树），解决了 React 重渲染导致连接中断的问题。

### 2.3 SFTP 文件管理器

**目标**：内置轻量级的图形化 SFTP 操作界面，免去用户切换额外工具。

| 功能 | 优先级 | 描述 |
|---|---|---|
| 目录浏览 (File Listing) | P0 | 带图标的树形文件列表，区分目录、文件、软链接 |
| 文件上传/下载 | P0 | 支持拖拽上传，`fastGet/fastPut` 流式传输 |
| 快速路径跳转 | P1 | 点击地址栏输入绝对路径一键跳转 |
| 文件实时同步编辑 | P1 | 双击文件在本地编辑器打开，本地保存时自动触发 SFTP 上传同步 |
| 新建文件/文件夹 | P1 | 内置 Modal，不依赖系统原生 `window.prompt` |
| 文件重命名 | P1 | 内联编辑 |
| 文件权限修改 (chmod) | P2 | 图形化权限选择 |
| 软链接 (Symlink) 支持 | P2 | 识别符号链接并支持穿透访问 |

**技术约束**：
- 文件大小须有上限校验，防止全量 `readFile` 导致的 OOM 崩溃（C-06 已记录）
- 临时文件名须使用随机 UUID 后缀，防止 TOCTOU 符号链接竞争

### 2.4 Command Center（主控台）

**目标**：提供一个统一的"入口门户"，整合连接、工具、插件于一身，体验对标 Raycast/Spotlight。

| 功能 | 优先级 | 描述 |
|---|---|---|
| 实时搜索过滤 | P0 | 模糊匹配所有已保存会话（支持 alias/host/username）|
| 键盘全流程导航 | P0 | ↑/↓ 选择条目，Enter 一键连接，ESC 关闭面板 |
| 快速连接入口 | P0 | 搜索框直接输入 `user@host` 回车，无需预先保存 |
| 插件入口聚合 | P1 | 「本地工具与插件」区域列出已安装插件，一键打开 |
| 仪表盘时钟 | P2 | 右上角显示实时日期时间，跟随 i18n 语言本地化 |
| 快捷操作丸 (Pill) | P2 | 快速跳转「新建连接」「设置」入口 |

### 2.5 凭证安全保险箱

**目标**：在本地持久化存储敏感凭证（密码、私钥路径）时，提供 OS 级加密保护。

| 功能 | 优先级 | 描述 |
|---|---|---|
| 主密码机制 | P0 | 用户首次使用须设置主密码（最低 8 字符）|
| 基于 Electron SafeStorage 加密 | P0 | macOS Keychain / Windows DPAPI / Linux Secret Service 驱动 |
| 生物识别解锁 | P1 | macOS 使用 Touch ID / Windows Hello 解锁主密码（须弹出系统确认框）|
| AES-256 + PBKDF2 | P0 | 主密码通过 100,000 轮 PBKDF2 派生，对会话凭证进行 AES-256-GCM 加密 |
| 安全模式 (Safe Mode) | P2 | 应用重启至安全模式时，禁用所有插件加载 |

---

### 2.6 插件系统

**目标**：提供一个受控、隔离的插件扩展生态，允许第三方开发者在不破坏核心安全边界的前提下扩展 GETSSH 功能。

| 功能 | 优先级 | 描述 |
|---|---|---|
| 插件安装 (ZIP 包) | P0 | 管理员从设置页面安装插件压缩包 |
| 插件卸载 | P0 | 一键卸载并清理残留文件 |
| 主进程插件 (Node.js) | P0 | 插件可访问受限的主进程 API（通过 Plugin SDK 白名单控制）|
| 渲染器插件 (iframe) | P0 | 插件 UI 在严格沙盒化的 iframe 内渲染（禁用 `allow-same-origin`）|
| RASP 沙盒防护 | P0 | 运行时通过 Proxy 拦截所有 `fs`/`child_process` 调用，阻断未授权访问 |
| 插件 Marketplace UI | P1 | 已安装插件列表，带版本号与描述 |
| `getssh-plugin://` 自定义协议 | P1 | 主进程托管插件静态资源的安全分发协议 |

**安全约束（硬性门槛）**：
- 插件 ZIP 包须通过 Zip Slip 防御验证
- 所有路径须通过 `getSecurePluginPath` 边界校验
- 插件不得调用 `safeStorageDecrypt`

### 2.7 安全中心 (SecureCenter / RASP)

**目标**：为用户提供可感知的实时运行时安全防护，而非隐形黑盒。

| 功能 | 优先级 | 描述 |

| 原型链污染检测 | P0 | 监控 `Object.prototype.__proto__` 等危险写入 |

| 安全告警 Overlay | P0 | 在渲染层显示红色安全警报弹窗，附带人类可读的风险描述 |
| 开发者模式豁免 | P1 | dev 模式下不触发 RASP 告警（供开发者调试）|
| 安全告警多语言 | P1 | 告警文案须本地化，避免英语技术术语吓到普通用户 |

## 3. 非功能性需求

### 3.1 性能

- 应用冷启动 < 3 秒
- 分屏状态切换无明显重连或卡顿
- 插件加载不阻塞主进程（全异步 I/O）

### 3.2 安全

- **CRITICAL 级漏洞 Zero Tolerance**：发布前必须修复所有 CRITICAL 级别漏洞
- IPC 通信须校验 `event.senderFrame` 来源身份
- CSP 须移除 `unsafe-eval`（允许保留 `unsafe-inline` 过渡期）

### 3.3 打包与分发

- macOS 包体积目标：≤ 85MB（通过 ULFO 压缩 + 极限 files 剔除）
- 支持后台静默更新检查（GitHub Releases API），检测到新版本时侧边栏设置图标显示红点

## 6. 产品设计哲学

> **「机制即防御，而非单靠加密」**
> 安全性应通过架构机制来保障，而非单纯依赖通信加密。

> **「开发者工具首先应该是工具，其次才是视觉体验」**
> UI 设计以极简为基调，在不破坏信息密度的前提下融入精致的细节动效。

> **「RASP 的告警应让人读懂，而非吓到人」**
> 安全告警应附带人类可读的风险描述，用母语告知用户潜在威胁。
